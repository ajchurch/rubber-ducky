# A5. Discover cryptographic implementation used online.

## End-To-End Encryption (E2EE)

A very commonly implemented use of cryptography online is something called End-To-End encryption. E2EE is essentially a type of messaging where messages are kept private except between each 'ends' (the person sending, and the person receiving). Only the parties involved in each 'end' of the messaging should be able to see the message (not even the platform communicated can see it). 

### What type of encryption does E2EE use?

E2EE typically uses asymmetric encryption which is essentially public key encryption enables two parties to communicate without having to send the secret key over an insecure channel. (CloudFlare, 2024)

Public key encryption relies on using two keys instead of one: a public key and a private key. While anyone, including the messaging service, can view the public key, only one person knows the private key. Data encrypted with the public key can only be decrypted with the private key (not the public key). This contrasts with symmetric encryption, where only one key is used to both encrypt and decrypt. (CloudFlare, 2024)

## WhatsApp

WhatsApp, a messaging app owned by Meta, uses End-To-End encryption when chatting to users on its platform. WhatsApp say, "..keeps your personal messages and calls between you and the person you’re communicating with. No one outside of the chat, not even WhatsApp, can read, listen to, or share them." 

They also have ways which you can test whether the encryption between you and another user are valid, either through automatic verfication, a QR code, or by comparing a 60 digit number.
