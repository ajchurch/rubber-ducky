# A8. Discover cryptography used in Internet of Things devices.

IoT devices discover and apply cryptography across several layers of their operation. To protect data in transit between devices, gateways, and cloud platforms, protocols like TLS is used, providing confidentiality through encryption, integrity through message authentication codes, and mutual authentication between device and server. For data stored locally (e.g., sensor readings, and credentials), symmetric algorithms like AES are typically used to guard against physical tampering or theft.

Device identity is established by either pre-shared keys that were put on it when it was made or digital certificates that use public key cryptography. Some devices use dedicated security chips to keep keys safe, so they can't be exposed to the rest of the system. It can be hard to figure out which method a device really uses because manufacturer documentation is often incomplete or misleading. To do this, you might need to capture its network traffic, extract and analyse its firmware, or probe its hardware interfaces. Even though there are well-known cryptographic tools, it is still very hard to use them consistently across the significant amount of different IoT devices in the world today.
