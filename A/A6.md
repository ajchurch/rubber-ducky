## Authenticator Apps and TOTPs (Time-based One-Time Passwords)

Authenticator apps are probably one of the most commonly used offline cryptographic implementations in the modern day. With the rise of 2FA and MFA to improve the security of your accounts, these apps have become vital in being the extra step in keeping accounts secure.

### How do they work?

While Authenticator apps initially require an internet connection to set up. They work entirely offline, doing maths locally on the device for whenever you need your 2fa code which is then compared to the server which is also doing those same calculations.

The authenticator apps use a TOTP algorithm which calculates the (usually 6 digit) code based on the current time and a shared secret between the app and the service you are logging into.

This [research paper](https://www.rfc-editor.org/rfc/pdfrfc/rfc6238.txt.pdf) by David M'Raihi , Johan Rydell , Mingliang Pei , and Salah Machani, helps to fully explain the algorithm and how TOTP works, and the below code is roughly what the algorithm looks like. (Samal, 2023)

```bash
class TOTP {

private static final int[] DIGITS_POWER
      // 0 1  2   3    4     5      6       7        8
      = {1,10,100,1000,10000,100000,1000000,10000000,100000000 };

public static String generateTOTP(String key,
                                    String time,
                                    String returnDigits,
                                    String crypto){
    int codeDigits = Integer.decode(returnDigits).intValue();
    String result = null;

    // Using the counter
    // First 8 bytes are for the movingFactor
    // Compliant with base RFC 4226 (HOTP)
    while (time.length() < 16 )
      time = "0" + time;

    // Get the HEX in a Byte[]
    byte[] msg = hexStr2Bytes(time);
    byte[] k = hexStr2Bytes(key);

    byte[] hash = hmac_sha(crypto, k, msg);

    // put selected bytes into result int
    int offset = hash[hash.length - 1] & 0xf;
     
    // bit masking to get unsigned integer
    int binary =
        ((hash[offset] & 0x7f) << 24) |
            ((hash[offset + 1] & 0xff) << 16) |
            ((hash[offset + 2] & 0xff) << 8) |
            (hash[offset + 3] & 0xff);

    int otp = binary % DIGITS_POWER[codeDigits];

    result = Integer.toString(otp);
    while (result.length() < codeDigits) {
      result = "0" + result;
    }
    return result;
  }

  public static void main(String[] args) {

    // Secret for HMAC-SHA256 - 32 bytes
    String seed32 = "313233343536373839303132333435363738"+
                    "3930313233343536373839303132";

    long T0 = 0;  // Start Time
    long X = 30; // Time for which the OTP should be valid
    long currentTime = 1234567890L;  // 2009-02-13 23:31:30 (UTC)

    long T = (currentTime - T0) / X;

    String steps = Long.toHexString(T).toUpperCase();

    final String otp = generateTOTP(seed32, steps, "6", "HmacSHA256");
    System.out.println("OTP is "+otp + " using | SHA256 |");

  }
}
```
